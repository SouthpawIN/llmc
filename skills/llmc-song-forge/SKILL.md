# LLMC Song Forge v3 — Wayne ✕ Mc.Baldiee ✕ Sleep Token

Three-register rap pipeline: Lil Wayne C3 swagger + Mc.Baldiee cosmic prose + Sleep Token worship vulnerability. Every song at 3:33 with ffmpeg trim.

## Architecture

```
Lyric DNA (3 registers → ORIGINAL metaphors, NEVER copy lines) → ACE-Step /release_task
  → Cache-watch WAV (48kHz) → HARD byte-verify duration → ffmpeg trim to 3:33 → MP3 → Discord push
```

## Lyric Diversity — Rotation Rules (MANDATORY)

**NEVER repeat the same opener or emotional position in a single batch.**
ACE-Step will pick up on repeated patterns and all tracks will sound samey.

### 5 Opener Types (ROTATE EVERY SONG)

1. **Comparison opener:** "I'm tougher than ___" — Wayne's signature, use sparingly (max 1 per batch)
2. **Story opener:** "You ever drag a body from the water..." — narrative hook, draws listener in
3. **Legacy opener:** "My grandmother used to say..." — generational wisdom, grounds the track
4. **Definition opener:** "I used to think love was addition..." — reframe a concept, intellectual hook
5. **Accusation opener:** "You spoke and I repeated you back..." — confrontational, immediate tension

### 5 Emotional Positions (ROTATE EVERY SONG)

1. **Triumphant:** "Now I'm ___ and you're still ___." — the winner, looking back
2. **Weary-then-rising:** "I used to ___ / now I ___." — exhaustion becoming power
3. **Confrontational:** "You think ___ / but ___." — calling out, naming names
4. **Reflective:** "Funny how ___." — the hard-won observation, the rueful smile
5. **Peaceful:** "The house is quiet now and ___." — resolution, calm, the after

## ACE-Step Generation Pipeline (CRITICAL — read before generating)

### Prerequisites
- ACE-Step API running at `http://127.0.0.1:8001`
- LM initialized (`llm_initialized: true` in /health)
- GPU with ~9GB+ free VRAM
- ffmpeg installed for trim + convert

### THE RULE: Duration Control (Two Verified Paths)

**`audio_duration` is a CAP, not a target.** ACE-Step generates audio until it runs out of lyrical content (ACE-Step discussion #95). Two paths to hit 3:33:

| Path | thinking | Speed | Mechanism | Best for |
|------|----------|-------|-----------|----------|
| **A: LM Planner** | `true` | ~120s | 5Hz LM plans duration via CoT, respects `audio_duration` | Small batches, unsure about metadata |
| **B: Dense Lyrics** | `false` | ~15s | 30-40 lyric lines → ~213s of audio (no LM) | Batch generation, speed matters |

**LINE-TO-DURATION MAP (with thinking:false):**
- 15 lines → ~86 seconds (1:26) — TOO SHORT
- 25 lines → ~160 seconds (2:40) — TOO SHORT
- 30+ lines → ~213 seconds (3:33) ✅
- 40+ lines → ~270+ seconds (4:30+)

Path B (thinking:false + 30+ lines) is verified faster and just as accurate. Always verify: `(stat_bytes - 44) / 192000 >= 200`. If <200, lyrics are too short.

### Pipeline Steps

```python
# 1. Post with thinking=true — LM handles duration
payload = {
    "prompt": "<detailed caption: style + instruments + texture + era + production + vocals + rhythm>",
    "lyrics": "<lyrics with structural tags AND modifiers: [Chorus - anthemic], [Bridge - whispered]>",
    "thinking": True,            # *** MANDATORY — LM plans duration ***
    "use_format": False,
    "bpm": 79,                   # Vary per track, NEVER repeat BPM+key in a batch
    "key_scale": "F# minor",
    "time_signature": "4",       # 4/4 standard; 6/8 for ballad/cinematic closers
    "vocal_language": "en",
    "inference_steps": 50,       # FULL QUALITY — 8 = turbo = silence
    "guidance_scale": 7.0,
    "audio_duration": 213,       # LM will honor this with thinking=true
    "audio_format": "wav",
    "task_type": "text2music",
}

# 2. Cache-watch for output WAV
baseline = {p.name for p in CACHE.glob("*.wav")}
# ... post task ...
new = {p.name for p in CACHE.glob("*.wav")} - baseline
if new:
    newest = sorted(CACHE.glob("*.wav"), key=lambda p: p.stat().st_mtime)[-1]

# 3. HARD VERIFY — don't trust ffprobe, trust the math
# ACE-Step outputs 48kHz stereo 16-bit WAV
# 213s × 48000Hz × 2ch × 2bytes = 40,896,000 bytes (plus 44-byte WAV header)
raw_bytes = path.stat().st_size - 44
actual_dur = raw_bytes / 192000  # bytes ÷ (48000 × 2 × 2)
if abs(actual_dur - 213) > 5:
    print(f"*** SHORT: {actual_dur:.1f}s — regenerate! ***")
```

### CRITICAL: 3:33 Verification

ACE-Step outputs at **48kHz** stereo 16-bit, not 44.1kHz. The math:
```
40,896,078 bytes total - 44 byte header = 40,896,034 audio bytes
40,896,034 ÷ (48000 × 2 × 2) = 213.0 seconds = 3:33 ✓
```

If the file is significantly smaller, the track is SHORT and needs regeneration with longer lyrics or higher BPM.

Always post-trim with ffmpeg to guarantee container is 213s:
```bash
ffmpeg -y -i input.wav -t 213 -codec:a libmp3lame -b:a 192k output.mp3
```

## Pitfalls (CRITICAL — every one of these was a user correction)

### 1. NEVER Copy Lines From Influence Artists

User explicitly and repeatedly called this out. Create ORIGINAL metaphors inspired by the style, not lifted from the source.

| Influence | DON'T Copy | DO Adapt Original |
|-----------|-----------|-------------------|
| Wayne simile | "tougher than Nigerian hair" | "tougher than crabgrass in a drought" |
| Wayne punchline | "venereal disease through the pencil" | "I was the blacksmith and the customer and the clerk" |
| Sleep Token body | "body, flesh, and bone" | "copper wire wrapped around your signal" |
| Sleep Token return | "take me back to Eden" | "the forest regrows after fire" |
| Sleep Token insect | "winged insect, funeral pyre" | "furniture rearranged in a burning room" |
| Sleep Token made | "when we were made it was no accident" | "the blueprint I drew had my fingerprints alone" |

**Rule: If the exact phrase appears in the source track, DON'T USE IT. Adapt the technique, not the words.**

### 2. thinking=false Breaks Duration

`thinking: false` skips the 5Hz LM planner, ACE-Step ignores `audio_duration`, and the DiT generates whatever block size it wants (usually 231s). ALWAYS use `thinking: true` for correct 3:33 duration. Cost: ~120s per track instead of ~20s. Worth it.

### 3. "I'm tougher than" Crutch

Don't open every song with it. The user noticed when multiple tracks had the same opener. Rotate the 5 opener types every song in a batch. Use comparison opener ("tougher than") at most ONCE per batch.

### 4. Repetitive Production

Don't use identical style prompts. Rotate BPM (68-90), key (different minor scales), and sub-genre (southern bounce → boom bap → melodic trap → cinematic) EVERY track. User noticed when beats sounded samey.

### 5. Discord Routing: MEDIA: NOT hermes send

`hermes send --to discord:<id>` sends to the HOME channel, never the current thread. Use `MEDIA:/absolute/path/to/song.mp3` inline in your response to land audio in the thread. User asked 3+ times before this was discovered.

### 6. ffprobe Lies; Bytes Don't

ffprobe reports container duration which can disagree with actual audio data. Always verify with raw byte math:
```python
raw_bytes = path.stat().st_size - 44  # strip 44-byte WAV header
actual_dur = raw_bytes / 192000  # 48kHz stereo 16-bit
```
If `actual_dur < 208`, the track is SHORT. Regenerate with longer lyrics or higher BPM.

### 7. ACE-Step Output is 48kHz

Not 44.1kHz as commonly assumed. All duration math must use 48kHz sample rate. 44-byte WAV header, 192,000 bytes/sec.

### 8. ACE-Step Caption Format

From official docs: Caption is THE most critical input. Be specific: style + instruments + emotion + timbre + era + production + vocals + rhythm + structure. Use structure tags WITH modifiers: `[Chorus - anthemic]`, `[Bridge - whispered]`, `[Verse 1 - building]`. 6-10 syllables per line with matched line lengths. ONE core metaphor per song.
